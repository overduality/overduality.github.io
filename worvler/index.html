A React web application implementing Shannon entropy-based optimal Wordle strategy through Web Workers and precomputed pattern matrices, achieving 100% solve rate with 3.62 average guesses in 13 games.

What This System Delivers

A production-grade Wordle assistant computing optimal guesses via Shannon's information entropy. Features two modes: Strategic (maximum information gain using "burner words") and Greedy (only guesses solutions). The Problem: Average players take 4-5 guesses using intuition and trial-and-error. Optimal strategy requires evaluating 13,000+ words per turn—impossible for humans, computationally expensive for naive implementations.

The Solution: Apply information theory to mathematically determine which guess maximizes expected information gain. Precompute all feedback patterns, use Web Workers for parallelization, deliver recommendations in <1 second.

Problem Statement

The Wordle Challenge

  • Solution Space: 2,309 possible answer words
  • Guess Space: 12,972 valid dictionary words
  • Feedback: 3⁵ = 243 possible patterns per guess
  • Goal: Identify solution in ≤6 guesses

Why Intuition Fails

  1. Random Guessing: "FUZZY" has low letter frequency—wastes information
  2. Greedy Local Search: Always guessing solutions leads to endgame traps (MATCH/BATCH/HATCH)
  3. No Lookahead: Humans can't evaluate 13,000 × 2,309 = 30M combinations

Information-Theoretic Insight

Best guess = word that maximizes expected information gain:

H(X) = -Σ p(x) × log₂(p(x))

Higher entropy = better partitioning → faster convergence.

Why TARSE Beats ADIEU

TARSE (5.95 bits) partitions solutions more evenly than ADIEU (4.82 bits). TARSE eliminates 81% of possibilities per guess vs ADIEU's 71%, that's 10% more information!

Information Theory Foundation

Shannon Entropy: Mathematical Definition

H(X) = -Σᵢ p(xᵢ) × log₂(p(xᵢ))

In Wordle context:

  • X = Feedback pattern (243 possibilities)
  • p(xᵢ) = Probability of pattern i
  • H(X) = Expected information gain in bits

Entropy Calculation

function calculateEntropy(guessWord, possibleSolutions) {
  const patternCounts = new Map();
  
  // Count patterns
  for (const solution of possibleSolutions) {
    const pattern = getFeedbackPattern(guessWord, solution);
    patternCounts.set(pattern, (patternCounts.get(pattern) || 0) + 1);
  }
  
  // Calculate entropy
  let entropy = 0;
  const total = possibleSolutions.length;
  
  for (const count of patternCounts.values()) {
    const p = count / total;
    entropy -= p * Math.log2(p);
  }
  
  return entropy;
}

Base-3 Encoding: Gray=0, Yellow=1, Green=2

Pattern = Σᵢ feedback[i] × 3ⁱ (fits in 1 byte)
